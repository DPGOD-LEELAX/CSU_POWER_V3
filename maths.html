§2

Load Modeling — P, Q, and S

Each bus has a rated apparent power (kVA). The simulator converts this into active power P (kW) and reactive power Q (kVAR) using the transformer power factor assumption. Load multipliers scale these values at runtime.

Active Power at Each Bus
Pi = kVAi × pftx × LMglobal × LMmeter
pftx = 0.9 (transformer power factor, fixed)
LMglobal = Global load multiplier (slider, 0.5 to 2.0)
LMmeter = Per-meter multiplier for M1, M2, or M3 (distributed only)
Eq. 2.1
Reactive Power at Each Bus
Qi = kVAi × √(1 − pftx²) × LM
Derived from the Pythagorean identity for power triangle: S² = P² + Q²
Eq. 2.2
Apparent Power (Magnitude)
|Si| = √(Pi² + Qi²) / 1000    (in kVA)
Eq. 2.3
Important distinction: The load multiplier (LM) scales power demand (P and Q), not the impedance. If you double the load, you double P and Q — but the line resistance R stays the same (unless you also change the impedance multiplier, §3).

Meter 1 (M1-A1 to M1-A6): 50 kVA each × 6 = 300 kVA
Meter 2 (M2-B1 to M2-B3): 100 kVA each × 3 = 300 kVA
Meter 3 (M3-C1 to M3-C3): 50 kVA each × 3 = 150 kVA
Total Distributed: 750 kVA

Centralized buses (kVA)

All served by a single 1500 kVA meter (Catgelo 1)
Delta connection — no neutral conductor

§3

Line Impedance & Impedance Multiplier

Each branch in the network has a complex impedance Z = R + jX, where R is resistance (causes real power loss) and X is reactance (causes reactive power loss). The Impedance Multiplier (Z_mult) scales both R and X proportionally.

Branch Impedance with Multiplier
Reff = Rbase × factori × Zmult
Xeff = Xbase × factori × Zmult
|Zeff| = √(Reff² + Xeff²)
factori is a per-branch scaling factor that represents the physical cable length/distance difference between branches (0.3 to 0.75).
Zmult is the user-adjustable global impedance multiplier (0.5× to 3.0×).
Eq. 3.1–3.3
ParameterDescriptionBase Value
RM1Resistance – Meter 1 branches0.0114 Ω
XM1Reactance – Meter 1 branches0.0473 Ω
RM2Resistance – Meter 2 branches0.0055 Ω
XM2Reactance – Meter 2 branches0.0226 Ω

XM3Reactance – Meter 3 branches0.0046 Ω
RcentResistance – Centralized main0.0158 Ω
XcentReactance – Centralized main0.0656 Ω
X/R ratio: For all branches, X/R ≈ 4.15. This is typical of medium-voltage distribution cables where reactance dominates. High X/R ratios mean voltage drop is more sensitive to reactive power (Q) than to active power (P).
§4

Network Topology — Radial Tree

CSU-Carig's distribution network is radial — there is exactly one path from the source (CAGELCO feeder) to each load bus. This makes BFS (Backward-Forward Sweep) the ideal solution method, since the network has no loops.

Distributed topology

Source (Node 0) → 3 parallel branches to Meters 1, 2, 3
Meter 1 (Node 1) → 6 load buses (Nodes 4–9)
Meter 2 (Node 2) → 3 load buses (Nodes 10–12)
Meter 3 (Node 3) → 3 load buses (Nodes 13–15)
Total: 16 nodes, 15 branches

Centralized topology

Source (Node 0) → Main Tx → Catgelo 1 meter (Node 0)
Catgelo 1 → Bus 1 (Node 1) → ... → Bus 12 (Node 12)
Feeder runs as a chain through all 12 buses
Total: 13 nodes, 12 branches

Branch Current Injection (Load Current)
Iload,i = |Si| / max(0.7, Vi)
Si is in kVA divided by 1000. The denominator uses max(0.7, Vi) to prevent division by near-zero voltage during early iterations when voltages haven't converged yet.
Eq. 4.1
§5

Backward-Forward Sweep (BFS) Method

The BFS method is specifically designed for radial (tree-like) distribution networks. It iterates in two phases until the solution converges. It is preferred over Newton-Raphson for radial networks because it is simpler, more numerically stable, and converges quickly.

0
Initialize:

Set all bus voltages to 1.0 pu: V[i] = 1.0 for all i. Set slack bus V[0] = 1.0 pu (fixed throughout). Set all branch currents I[j] = 0.

1
Compute load currents at each bus:

For each load bus i, calculate Iload,i = |Si| / max(0.7, V[i]), where Si depends on Pi, Qi, and the current iteration's voltage.

2
Backward Sweep — sum currents from leaves to root:

Traverse the tree from each leaf bus toward the source. For each branch, accumulate: Ibranch,j = Iload,child + Σ(currents from all children of child). This gives the total current flowing through each branch.

3
Forward Sweep — compute voltages from root to leaves:

Starting from the slack bus, propagate voltages forward. For each branch from node m to node n: V[n] = V[m] − (I[j] × |Zj| × dropScale)

4
Check convergence:

Compute max deviation: Δ = max|Vnew[i] − Vold[i]|. If Δ < 0.0001, stop. Otherwise, store Vold = Vnew and go to Step 1.

Backward Sweep — Branch Current Accumulation
Ij = Iload,child(j) + Σ Ik   (k = all branches downstream of child(j))
Eq. 5.1
Forward Sweep — Voltage Update
Vn = Vm − Ij × |Zj| × dropScale
Distributed system: dropScale = 3.5 (accounts for three-phase and feeder characteristics)
Centralized system: dropScale = 0.35 (single main feeder with shorter runs)
Vn is clamped: max(0.70, Vn) to prevent nonphysical values.
Eq. 5.2
Convergence Criterion
Δmax = maxi |Vi(k) − Vi(k−1)| < 0.0001 pu
Eq. 5.3
§6

Voltage Drop Calculation

Voltage drop is the key performance metric for distribution systems. The Philippine Electrical Code (PEC) 2025 allows a maximum of 5% voltage drop from the source to any load terminal.

Voltage Drop (Per Unit)
ΔVpu = Vsource − Vbus,i
Eq. 6.1
Voltage Drop (Percentage)
ΔV% = (1.0 − Vbus,i) × 100
Since source = 1.0 pu, the drop is simply (1 − Vpu) × 100%
Eq. 6.2
Voltage Drop (in kV, at 13.7 kV base)
VkV = Vpu × 13.7 kV
Eq. 6.3
Voltage Angle (approximate)
δi = −(1.0 − Vi) × Kangle
Kangle = 20 for distributed, 30 for centralized.
A larger angle indicates greater phase displacement from the slack bus, proportional to voltage drop.

§7

Loss Calculations

The simulator computes three types of losses. Each has a different cause and behavior with respect to load and system parameters.

7.1 — Copper Losses (I²R)

These are real power losses that occur whenever current flows through the resistance of a conductor. They are the dominant loss component and increase with the square of current.

Copper Loss — per branch
Ploss,j = Ij² × Rj × 1000   (kW)
Multiplied by 1000 to convert from per-unit watts to kW. Ij and Rj are the current and resistance of branch j.
Eq. 7.1
Reactive Loss — per branch
Qloss,j = Ij² × Xj × 1000   (kVAR)
Eq. 7.2
Total Copper Loss (all branches)
Pcopper = Σj Ij² × Rj × 1000
Eq. 7.3

7.2 — Core (Iron) Losses

These are constant losses in transformer cores caused by magnetic hysteresis and eddy currents. They are independent of load — they exist even when the transformer is on with no current flowing.

Core Loss
Pcore = Capacitytotal × kcore
kcore = 0.005 (0.5% of rated transformer capacity, a standard engineering approximation)
Distributed: 750 kVA × 0.005 = 3.75 kW
Centralized: 1500 kVA × 0.005 = 7.5 kW
Eq. 7.4

7.3 — Harmonic Losses (THD Effect)

Non-linear loads (computers, LED drivers, variable-frequency drives) inject current harmonics into the network. Harmonic currents flow at multiples of 50 Hz (100 Hz, 150 Hz, ...) and increase conductor heating beyond what the fundamental frequency current would cause.

Harmonic Multiplier
Kh = 1 + (THD% / 100) × 2
This factor accounts for increased conductor losses due to harmonic currents. At THD = 3%: Kh = 1.06 (6% more losses than fundamental alone).
Eq. 7.5
Total Harmonic Loss
Pharmonic = Pcopper × (Kh − 1) + (Pload × 0.005 × (THD% / 5))
First term: extra copper heating from harmonics (proportional to existing I²R loss).
Second term: direct load-side harmonic losses from non-linear equipment.
Eq. 7.6
Total System Loss
Ptotal loss = Pcopper + Pcore + Pharmonic
Loss% = (Ptotal loss / Pload) × 100
Eq. 7.7 – 7.8
§8

PEC 2025 Compliance Check

The Philippine Electrical Code 2025 mandates that voltage at any load terminal shall not fall below 95% of the nominal voltage (i.e., ≥ 0.95 pu). The simulator checks this automatically after each BFS solution.

Worst-case voltage drop
Vworst = mini(Vi)    across all load buses
ΔV%worst = (1.0 − Vworst) × 100
Eq. 8.1 – 8.2
PEC Compliant: System passes if Vworst ≥ 0.95 pu (ΔV% ≤ 5.00%). At 13.7 kV base, this means the minimum acceptable bus voltage is 13.015 kV.
PEC Violation: System fails if any bus voltage falls below 0.95 pu. In practice this means loads experience undervoltage, which can damage motors, cause equipment malfunction, and reduce efficiency.

§10

Summary of All Equations

Quick-reference table of every equation used in the simulator, in order of computation.

Step 1 — Load Power
Pi = kVAi × pftx × LMglobal × LMmeter Active power (kW) per bus
Qi = kVAi × √(1 − pftx²) × LM Reactive power (kVAR) per bus
|Si| = √(Pi² + Qi²) / 1000 Apparent power (kVA)
Step 2 — Impedance
Reff = Rbase × factori × Zmult Effective resistance (Ω)
Xeff = Xbase × factori × Zmult Effective reactance (Ω)
|Zj| = √(Rj² + Xj²) Impedance magnitude (Ω)
Step 3 — BFS Load Current
Iload,i = |Si| / max(0.7, Vi) Current injection at each load bus
Step 4 — Backward Sweep
Ij = Iload,child + Σ Idownstream Branch current (accumulate leaf → root)
Step 5 — Forward Sweep
Vn = Vm − Ij × |Zj| × dropScale Bus voltage update (root → leaves)
Step 6 — Convergence Check
Δmax = max|V(k) − V(k−1)| < 0.0001 pu Stop if converged
Step 7 — Losses
Ploss,j = Ij² × Rj × 1000 Copper loss per branch (kW)
Pcore = Capacitytotal × 0.005 Core (iron) loss (kW)
Kh = 1 + (THD% / 100) × 2 Harmonic multiplier
Pharmonic = Pcopper × (Kh−1) + Pload × 0.005 × (THD%/5) Harmonic losses
Ptotal loss = Pcopper + Pcore + Pharmonic Total system losses (kW)
Step 8 — Compliance
ΔV% = (1.0 − Vworst) × 100 Worst-case voltage drop
Compliant if ΔV% ≤ 5.00% (i.e., Vworst ≥ 0.95 pu) PEC 2025 standard
The simulator uses the Backward-Forward Sweep method — an industry-standard power flow algorithm for radial distribution networks (documented in Kersting, W.H., "Distribution System Modeling and Analysis," 2002). It converges in fewer than 50 iterations with tolerance 0.0001 pu, making it computationally efficient for real-time interactive use.
